# Portfolio - Mehdi Rhifar

## Vue d'ensemble

Site web portfolio personnel destiné à être déployé sur **mehdirhifar.dev**. Ce projet présente l'expérience professionnelle, les compétences techniques, les projets et le parcours éducatif de Mehdi Rhifar.
Le but de mon site va être de me mettre en avant pour des possible recruteur, et avoir un crédibilité en tant que developpeur. Ca doit donc être un mix entre du détail et la simplicité de lecture.
Le site est construit avec **Astro 5.1.5**, un framework web moderne orienté contenu qui génère des sites statiques performants.

## Informations personnelles

- **Nom** : Mehdi Rhifar
- **Domaine cible** : mehdirhifar.dev
- **Profil** : Développeur fullstack qui préfère le backend
- **Spécialités** : Backend development

- **Langages** : TypeScript, Astro components
- **Styling** : CSS personnalisé (public/assets/styles.css, color-enhancements.css)

### Commandes disponibles

```bash
npm run dev      # Serveur de développement (http://localhost:4321)
npm run build    # Build de production (site statique)

```

## Architecture du projet

### Collections de contenu (src/content/)

Le projet utilise le système de collections Astro avec trois types de contenu :

#### 1. Projects (src/content/projects/)
Schéma :
- `title` : Nom du projet
- `shortDesc` : Tableau de descriptions courtes
- `tech` : Technologies utilisées (array)
- `category` : "personal" ou "professional"
- `featured` : Booléen pour les projets mis en avant
- `order` : Ordre d'affichage
- `links` : Objet contenant github, demo, npm (optionnels)

#### 2. Experiences (src/content/experiences/)
Schéma :
- `company` : Nom de l'entreprise
- `role` : Poste occupé
- `startDate` / `endDate` : Dates au format string
- `location` : Lieu
- `description` : Description du rôle
- `achievements` : Tableau des réalisations
- `skills` : Technologies et compétences (array)
- `order` : Ordre d'affichage

#### 3. Education (src/content/education/)
Schéma :
- `school` : Établissement
- `degree` : Diplôme
- `field` : Domaine d'études (optionnel)
- `startDate` / `endDate` : Dates
- `location` : Lieu
- `highlights` : Points marquants (array, optionnel)
- `order` : Ordre d'affichage
